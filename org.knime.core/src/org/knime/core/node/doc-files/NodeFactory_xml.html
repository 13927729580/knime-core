The XML file that must be provided with the node (or the NodeFactory). It descripes the node and its components as well as its ports - for the KNIME user.

Here is a nonsense example, containing all possible sections. For the explanation of  the sections (each marked with the corresponding number), please see below.

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE knimeNode PUBLIC "-//UNIKN//DTD KNIME Node 1.0//EN" "http://www.knime.org/Node.dtd">
<knimeNode (1)icon="./myIcon.png" (2)type="NodeType">
    (3)<name>MyNodeName</name>
    (4)<shortDescription>Whatever myNode does in one sentence.</shortDescription>
    (5)<fullDescription>
        (5.1)<intro>
        A description of the node's functionality and speciality.
        <p>You can create paragraphs and lists:
            <ol>
            <li>Firstly</li>
            <li>Secondly(Y axis)</li>
            <li>...</li>
            </ol>
        </p>
        For the full and exact specification of which tags are allowed in which
            section, please refer to the DTD file (located in the org.knime.core.node
            package or at the above URL.)</b></p>
      </intro>
        (5.2)<option name="Dialog option">
            If the node has a dialog, explain the options in the node's
            dialog here. Use names the user will be able to associate
            with the corresponding dialog option.
        </option>
      <option name="Dialog option2">
        Add as many option sections as needed.
        </option>
    </fullDescription>
    (6)<ports>
        <dataIn index="0" name="Left table">Descriptive text for the
               data coming through that port.</dataIn>
        <dataIn index="1" name="Right table">This text will be displayed
               in the view.</dataIn>
        <dataOut index="0" name="Joined table">While the 'name' is used
               in the menus.</dataOut>
        <predParamIn index="0" name="Model">More about this port.</predParamIn>
        <predParamOut index="0" name="Model">Description of the model
               coming through this port.</predParamOut>
    </ports>
    (7)<views>
        <view index="0" name="Award Winning View">
            Verbosely descripe the view here.
        </view>
        <view index="1" name="Next View">
            Add as many entries as you have views with that node.
        </view>
    </views>
</knimeNode>
If your node has no dialog, you probably won't have any <option></option> section.
If you have no view with your node, remove the <view></view> section.
Here is the description of the marked items above. If you copy&paste the above, remove the red numbers from the text.
  1. Specify your icon. If not set, the KNIME default icon will be used.
  2. Specify the type. Legal types are: Source, Sink, Learner, Predictor, Manipulator, Visualizer, Meta, or Other
  3. Provide a name in the <name> section. This name will be displayed in the node repository view, above the node, if it is dragged into workflow, and as the header in the node description view.
  4. The <shortDescription> section should contain a one liner (like 80 to 100 chars) saying in short what the node does. It will be used in the node description view when multiple nodes are selected.
  5. The <fulldescription> section contains an <intro> and optionally - possibly multiple - <option> sections. The options - if defined - will be listed in the node description view under "Dialog Options".
    • In the <intro> section you descripe the node's functionality. Explain when it does what. What it expects at its inports, which data will be delivered at its outports. You may also add references to external resources, like publications, algorithms, etc.
    • If the node has a dialog you should list all user adjustable settings in the list of options. For each option add an <option> section with a name (which will be the header for this item), descripe all possible settings and their consequences.
  6. In the <ports> section, for each port of the node, add one of the following:
    <dataIn index="" name="">description</dataIn>, for a data table input port,
    <dataOut index="" name="">description</dataOut>, for a data table output port,
    <predParamIn index="" name="" type="">description</predParamIn>, and
    <predParamOut index="" name="" type="">description</predParamOut>, for a content model input or output port.
    The index  must be the same index as used in the source code. The name provided will be displayed in the menus. The description (between the tags) shows up in the node description view. Model ports (for content/data models) have an optional 'type' attribute. The GUI will allow only to connect ports with matching type atributes. If the type is not specified, the port connects to any type (but still only to model ports).
  7. If your node comes with a, or multiple differnt views, add a <views> section. For each view, add a <view> tag, provide an index (the same as in the source code), and a name. The name will be shown in the menus, the descriptive text of the view tag, will be displayed in the node description view.
