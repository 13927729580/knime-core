The meta package contains all classes needed to build a meta node that encapsulates a meta workflow. We distinguish between MetaIONodes that are responsible for translating the input/output of the outer workflow to the inner workflow and the MetaNode itself that holds an internal workflow.

The following classes belong to the MetaNode:
  • MetaNodeContainer, MetaNode and MetaNodeModel
  • All MetaNodeFactories ending with the number of input and output ports
The MetaNode itself behaves like a normal node. The basic difference is that the workbench can access the inner workflow of this node. The MetaNodeContainer accesses the inner meta-workflow through the MetaNode which access the underlying MetaNodeModel. Input DataTables are translated from the inports of the MetaNode in the inner workflow and from its outputs to the outports of the MetaNode.
The following classes belong to the MetaIONodes:
  • MetaInputNodeModel, MetaInputNode and MetaInputNodeContainer
  • MetaOutputNodeModel, MetaOutputNode and MetaOutputNodeContainer
  • MetaIONode and MetaIONodeContainer

These nodes can only be used inside a MetaWorkflow. They are responsible for translating the input at the MetaNode to the inner workflow and resp. translating the output to the output port of the MetaNode. The MetaInput and MetaOutput nodes allow to build a fully connected workflow inside a MetaNode. The MetaInput- and MetaOutputNode behave like normal nodes. The only difference is that DataTables and DataTableSpecs are manually set resp. retrieved by getter- and setter-methods. DataTables and DataTableSpecs are propagated from the NodeContainer through the Node to the NodeModel. The Factory of the MetaInput and MetaOutputNode produces singletons.

If you want to write your own meta nodes, have a look at Howto write a meta node.
