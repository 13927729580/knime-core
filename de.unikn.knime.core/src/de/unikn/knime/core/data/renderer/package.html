Interfaces and default implementations for renderer of DataCell objects.

This package describes the general structure for renderer and renderer families of DataCell. Generally, each DataType implementation can define its own set of renderer. These renderer are accumulated in a DataCellRendererFamily.

Here is what you should do if you implement your own DataType and make one or more renderer available for it:

  • Create your own renderer implementing the DataCellRenderer interface. (You might consider to derive from DefaultDataCellRenderer).
  • Implement a DataCellRendererFamily that either uses singletons of the renderers above or creates them on request. As before: it is probably useful to extend the default implementation, i.e. DefaultDataCellRendererFamily.
  • Override in your DataType implementation the methode getNativeRenderer() and return your newly created renderer family.
